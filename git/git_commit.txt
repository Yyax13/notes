3✅ Padrão de mensagem de commit:
<tipo>(escopo): mensagem breve no imperativo

Exemplos:
feat(login): adiciona validação de senha fraca
fix(cadastro): corrige erro ao salvar novo cliente

Explicando os componentes:
Tipo     -> Define o tipo de mudança no código (ex: feat, fix, docs, etc).
Escopo   -> (Opcional) Área específica afetada (ex: login, produto, api).
Mensagem -> Descreve claramente a mudança feita, no imperativo ("adiciona", "corrige").

Tipos comuns de commit (<tipo>):
feat	 -> Nova funcionalidade
fix	     -> Correção de bug
docs	 -> Alterações na documentação
style	 -> Mudanças de estilo (espaços, identação, formatação)
refactor -> Refatorações (alterações internas que não mudam funcionalidade)
test	 -> Adição ou modificação de testes
chore	 -> Tarefas diversas (ex: atualizar dependências, configs, scripts)

Dicas Importantes:
 - Escreva a mensagem no imperativo (ex: "corrige bug", e não "corrigido bug").
 - Seja breve, mas claro.
 - Use português ou inglês conforme o padrão do projeto (mantenha consistência).
